## 1. 产品概述

用户权限管理系统是一套面向企业内部的后台管理基础平台，提供用户管理、角色管理、权限分配与访问控制等核心功能，可作为各类业务系统的认证与授权底座。
- **目标用户**：系统管理员、IT运维人员
- **核心价值**：通过角色-权限-用户三级模型实现灵活的访问控制，支持细粒度权限分配，保障系统安全与数据隔离

## 2. 核心功能

### 2.1 用户角色
| 角色 | 注册方式 | 核心权限 |
|------|---------|----------|
| 超级管理员 | 系统预置 | 全部模块访问、用户管理、角色管理、权限管理、系统配置 |
| 普通管理员 | 超级管理员创建 | 用户管理（查看/新增/编辑）、角色查看、权限查看 |
| 普通用户 | 管理员创建 | 仅查看个人信息、修改密码 |

### 2.2 功能模块
1. **登录认证**：账号密码登录、JWT Token 认证、登录状态持久化
2. **仪表盘**：用户总数、角色数量、在线用户数、最近登录记录
3. **用户管理**：用户列表、新增用户、编辑用户、删除用户、分配角色、重置密码、启用/禁用
4. **角色管理**：角色列表、新增角色、编辑角色、删除角色、分配权限
5. **权限管理**：权限树展示、权限分组、新增权限、编辑权限、删除权限
6. **个人中心**：个人信息查看、修改密码、操作日志

|---------|---------|---------|

## 3. 核心流程

**用户登录流程**：用户输入账号密码 → 后端校验 → 返回JWT Token → 前端存储Token → 跳转仪表盘 → 后续请求携带Token鉴权

**创建用户流程**：管理员进入用户管理 → 点击新增 → 填写用户信息与角色 → 提交 → 后端创建用户并关联角色 → 刷新用户列表

**权限分配流程**：管理员进入角色管理 → 选择角色 → 分配权限 → 勾选权限树节点 → 提交 → 后端更新角色权限关联 → 用户下次登录时权限生效


## 4. 用户界面设计

### 4.1 设计风格
- **主题方向**：经典后台管理风格，简洁稳重，以功能性为主导
- **主色**：深蓝 `#304156`（侧边栏与导航）/ 品牌蓝 `#409EFF`（按钮、链接、高亮）
- **辅色**：成功绿 `#67C23A`、警告橙 `#E6A23C`、危险红 `#F56C6C`、信息灰 `#909399`
- **背景**：主内容区浅灰 `#F0F2F5`，卡片白色 `#FFFFFF`
- **按钮风格**：圆角4px，主按钮品牌蓝填充，次按钮描边，文字按钮无边框
- **字体**：系统字体栈 `-apple-system, BlinkMacSystemFont, "Segoe UI", "PingFang SC", "Microsoft YaHei", sans-serif`
- **布局**：左侧深色固定侧边栏 + 顶部导航栏 + 主内容区（面包屑 + 内容卡片）
- **图标**：Element Plus 内置图标集
- **表单**：标签左对齐、输入框统一高度、校验提示明确

### 4.2 页面设计概览
| 页面名称 | 模块名称 | UI元素 |
|---------|---------|--------|
| 登录页 | 登录表单 | 居中卡片、系统Logo+标题、用户名密码输入框、登录按钮、背景渐变 |
| 仪表盘 | 统计卡片 | 四列网格、图标+数字+标签、hover微浮效果 |
| 仪表盘 | 最近登录 | 表格卡片、状态标签(成功/失败)、时间格式化 |
| 用户管理 | 用户列表 | 搜索栏+操作按钮栏+数据表格+分页器、状态开关、操作按钮组 |
| 用户管理 | 用户表单 | 弹窗Dialog、表单校验、角色多选下拉框 |
| 角色管理 | 角色列表 | 卡片网格布局、角色名称+标识+描述+用户数徽章、操作按钮 |
| 角色管理 | 权限分配 | 弹窗+树形控件、全选/展开/折叠、父子节点联级 |
| 权限管理 | 权限树 | 左侧树形控件+右侧详情面板、节点hover操作按钮、类型图标区分 |
| 个人中心 | 信息展示 | 卡片布局、描述列表、头像占位、修改密码独立表单 |

### 4.3 响应式
- 桌面优先（≥1280px）：完整侧边栏 + 多列布局
- 平板（768-1279px）：可折叠侧边栏 + 双列布局
- 移动端（<768px）：顶部导航栏 + 单列堆叠，表格支持横向滚动
